12 /*
13  * AP_InertialNav blends accelerometer data with gps and barometer data to improve altitude and position hold.
14  *
15  * Most of the functions have to be called at 100Hz. (see defines above)
16  *
17  * The accelerometer values are integrated over time to approximate velocity and position.
18  * The inaccurcy of these estimates grows over time due to noisy sensor data.
19  * To improve the accuracy, baro and gps readings are used:
20  * An error value is calculated as the difference between the sensor's measurement and the last position estimation.
21  * This value is weighted with a gain factor and incorporated into the new estimation
22  *
23  * Special thanks to Tony Lambregts (FAA) for advice which contributed to the development of this filter.
24  *
25  */
