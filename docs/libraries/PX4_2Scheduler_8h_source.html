25 /* how long to boost priority of the main thread for each main
26  loop. This needs to be long enough for all interrupt-level drivers
27  (mostly SPI drivers) to run, and for the main loop of the vehicle
28  code to start the AHRS update.
29 
30  Priority boosting of the main thread in delay_microseconds_boost()
31  avoids the problem that drivers in hpwork all happen to run right
32  at the start of the period where the main vehicle loop is calling
33  wait_for_sample(). That causes main loop timing jitter, which
34  reduces performance. Using the priority boost the main loop
35  temporarily runs at a priority higher than hpwork and the timer
36  thread, which results in much more consistent loop timing.
37 */
38 #define APM_MAIN_PRIORITY_BOOST_USEC 150
